# Status File Formats

This document defines the JSON formats for pipeline status tracking files generated during knowledge base initialization.

## Overview

Status files are generated by the dispatch skill after each pipeline stage completes. They provide:
- Execution tracking for parallel workers
- Progress reporting for downstream processes
- Audit trail for troubleshooting

## File Locations

| Pipeline | Stage | File Path |
|----------|-------|-----------|
| Bizs | Stage 2 | `speccrew-workspace/knowledges/base/sync-state/knowledge-bizs/stage2-status.json` |
| Bizs | Stage 3 | `speccrew-workspace/knowledges/base/sync-state/knowledge-bizs/stage3-status.json` |
| Techs | Stage 2 | `speccrew-workspace/knowledges/base/sync-state/knowledge-techs/stage2-status.json` |
| Techs | Stage 3 | `speccrew-workspace/knowledges/base/sync-state/knowledge-techs/stage3-status.json` |

## Bizs Pipeline Status Files

### Stage 2: Module Analysis

**Generation Timing**: After all Bizs Stage 2 Workers complete

**Schema**:
```json
{
  "generated_at": "ISO-8601 timestamp",
  "stage": "module-analysis",
  "total_modules": "number",
  "completed": "number",
  "failed": "number",
  "platforms": [
    {
      "platform_type": "string",
      "platform_subtype": "string (optional)",
      "platform_name": "string",
      "modules": [
        {
          "module_name": "string",
          "status": "completed | failed",
          "features_count": "number",
          "output_path": "string"
        }
      ]
    }
  ]
}
```

**Example**:
```json
{
  "generated_at": "2024-01-15T10:30:00Z",
  "stage": "module-analysis",
  "total_modules": 8,
  "completed": 8,
  "failed": 0,
  "platforms": [
    {
      "platform_type": "web",
      "platform_name": "Web Frontend",
      "modules": [
        {
          "module_name": "order",
          "status": "completed",
          "features_count": 5,
          "output_path": "speccrew-workspace/knowledges/bizs/web/order/"
        },
        {
          "module_name": "payment",
          "status": "completed",
          "features_count": 3,
          "output_path": "speccrew-workspace/knowledges/bizs/web/payment/"
        }
      ]
    },
    {
      "platform_type": "mobile",
      "platform_subtype": "flutter",
      "platform_name": "Mobile App",
      "modules": [
        {
          "module_name": "order",
          "status": "completed",
          "features_count": 4,
          "output_path": "speccrew-workspace/knowledges/bizs/mobile/order/"
        }
      ]
    }
  ]
}
```

## Techs Pipeline Status Files

### Stage 2: Platform Document Generation

**Generation Timing**: After all Techs Stage 2 Workers complete

**Schema**:
```json
{
  "generated_at": "ISO-8601 timestamp",
  "stage": "platform-doc-generation",
  "total_platforms": "number",
  "completed": "number",
  "failed": "number",
  "platforms": [
    {
      "platform_id": "string",
      "platform_type": "string",
      "framework": "string",
      "status": "completed | failed",
      "documents_generated": ["array of filenames"],
      "output_path": "string"
    }
  ]
}
```

**Example**:
```json
{
  "generated_at": "2024-01-15T10:30:00Z",
  "stage": "platform-doc-generation",
  "total_platforms": 3,
  "completed": 3,
  "failed": 0,
  "platforms": [
    {
      "platform_id": "web-react",
      "platform_type": "web",
      "framework": "react",
      "status": "completed",
      "documents_generated": [
        "INDEX.md",
        "tech-stack.md",
        "architecture.md",
        "conventions-design.md",
        "conventions-dev.md",
        "conventions-test.md"
      ],
      "output_path": "speccrew-workspace/knowledges/techs/web-react/"
    },
    {
      "platform_id": "backend-nestjs",
      "platform_type": "backend",
      "framework": "nestjs",
      "status": "completed",
      "documents_generated": [
        "INDEX.md",
        "tech-stack.md",
        "architecture.md",
        "conventions-design.md",
        "conventions-dev.md",
        "conventions-test.md",
        "conventions-data.md"
      ],
      "output_path": "speccrew-workspace/knowledges/techs/backend-nestjs/"
    }
  ]
}
```

### Stage 3: Root Index Generation

**Generation Timing**: After Techs Stage 3 Worker completes

**Schema**:
```json
{
  "generated_at": "ISO-8601 timestamp",
  "stage": "root-index-generation",
  "status": "completed | failed",
  "platforms_indexed": "number",
  "index_file": "string (path to INDEX.md)"
}
```

**Example**:
```json
{
  "generated_at": "2024-01-15T10:35:00Z",
  "stage": "root-index-generation",
  "status": "completed",
  "platforms_indexed": 3,
  "index_file": "speccrew-workspace/knowledges/techs/INDEX.md"
}
```

## Field Reference

### Common Fields

| Field | Type | Description |
|-------|------|-------------|
| `generated_at` | string | ISO 8601 timestamp when status was generated |
| `stage` | string | Pipeline stage identifier |
| `status` | string | "completed" or "failed" |

### Bizs Pipeline Fields

| Field | Type | Description |
|-------|------|-------------|
| `total_modules` | number | Total modules dispatched |
| `completed` | number | Successfully completed modules |
| `failed` | number | Failed modules |
| `platforms[].platform_type` | string | Platform type identifier |
| `platforms[].platform_subtype` | string | Platform subtype (optional) |
| `platforms[].platform_name` | string | Human-readable platform name |
| `platforms[].modules[].module_name` | string | Module code_name |
| `platforms[].modules[].features_count` | number | Number of features extracted (Stage 2 only) |
| `platforms[].modules[].output_path` | string | Module output directory (Stage 2 only) |
| `platforms[].modules[].overview_file` | string | Path to completed overview document (Stage 3 only) |

### Techs Pipeline Fields

| Field | Type | Description |
|-------|------|-------------|
| `total_platforms` | number | Total platforms dispatched |
| `completed` | number | Successfully completed platforms |
| `failed` | number | Failed platforms |
| `platforms[].platform_id` | string | Platform identifier |
| `platforms[].framework` | string | Framework name |
| `platforms[].documents_generated` | array | List of generated document filenames |
| `platforms_indexed` | number | Number of platforms in root index (Stage 3 only) |
| `index_file` | string | Path to generated INDEX.md (Stage 3 only) |
