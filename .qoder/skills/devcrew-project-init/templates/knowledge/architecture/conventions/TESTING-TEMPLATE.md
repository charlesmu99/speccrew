# 测试规范

> 使用者：开发 Agent（必须读）、测试 Agent（必须读）
> 维护者：amu-agent 初始化生成，测试策略变化时更新

---

## 测试策略

| 测试类型 | 覆盖目标 | 工具 | 覆盖率要求 |
|----------|----------|------|------------|
| 单元测试 | 函数/方法级别 | [工具] | [要求] |
| 集成测试 | 接口级别 | [工具] | [要求] |
| 端到端测试 | 用户流程级别 | [工具] | [要求，可选] |

## 单元测试规范

### 文件命名
```
# 前端
[组件名].test.ts   或  [组件名].spec.ts

# 后端
test_[模块名].py
```

### 测试用例命名
```python
# 后端示例
def test_[被测函数]_[场景]_[预期结果]():
    # 示例：
    # test_create_user_with_valid_data_returns_user_id
    pass
```

```typescript
// 前端示例
it('should [预期行为] when [条件]', () => {})
```

### 测试结构（AAA 原则）
```python
def test_xxx():
    # Arrange - 准备测试数据
    
    # Act - 执行被测逻辑
    
    # Assert - 验证结果
```

## 测试数据规范

- 测试数据不依赖外部状态
- 使用 fixture 或 factory 生成测试数据
- 测试结束后清理副作用（数据库、文件等）

## 不需要测试的内容

- 框架本身的功能
- 简单的数据传递（getter/setter）
- 第三方库的内部逻辑
