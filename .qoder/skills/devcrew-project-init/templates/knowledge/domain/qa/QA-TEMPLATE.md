# 常见问题与解决方案（QA）

> 使用者：所有 Agent（按需读）
> 维护者：遇到疑难问题解决后人工沉淀

---

## 开发踩坑记录

> 记录开发过程中遇到的陷阱、常见错误及解决方案，供后续 Agent 参考避免重复踩坑。

### 记录格式

每个踩坑记录按以下格式添加：

```markdown
### [问题简述]

**场景**：[什么情况下会遇到]
**现象**：[具体错误表现]
**根因**：[为什么会发生]
**解决方案**：[如何解决]
**预防措施**：[如何避免再次出现]
**记录日期**：[YYYY-MM-DD]
**记录人**：[Agent/人工]
```

### 示例

### Docker 服务启动顺序问题

**场景**：后端服务依赖数据库，但启动时数据库还未就绪
**现象**：`Connection refused` 错误，服务启动失败
**根因**：Docker Compose 默认并行启动，不保证依赖服务先就绪
**解决方案**：
1. 使用 `depends_on` 配合健康检查
2. 或在代码中加入重连机制
**预防措施**：新增服务时检查依赖关系，配置健康检查
**记录日期**：2026-03-14
**记录人**：Dev Agent
