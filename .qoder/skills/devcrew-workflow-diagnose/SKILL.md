# 触发场景

- AI 工程化流程在某个环节卡住
- Agent 输出不符合预期
- 产出物质量不达标
- 用户说"流程有问题"、"诊断一下"、"Agent行为异常"

# 工作流程

## 1. 收集问题信息

询问或识别以下信息：
- 问题发生在哪个环节？（PM → Solution → 设计 → 开发 → 测试）
- 具体表现是什么？
- 已尝试过哪些解决方法？

## 2. 读取相关产出物

根据问题环节，读取对应的产出物：

| 问题环节 | 需要读取的产出物 |
|----------|------------------|
| PM 阶段 | `se/prd.md` |
| Solution 阶段 | `se/solution.md` |
| 设计阶段 | `se/*-design.md` |
| 开发阶段 | 相关代码文件 |
| 测试阶段 | 测试用例、测试报告 |

## 3. 分析问题根因

常见问题类型及诊断方向：

### Agent 行为问题

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| 输出过于笼统 | Agent 提示词缺少项目具体信息 | 使用 `agent-optimize` Skill 补充具体技术栈 |
| 跳过必要步骤 | 工作流程描述不清 | 增加检查清单（checkbox） |
| 输出格式混乱 | 输出规范不明确 | 提供具体模板示例 |
| 职责越界 | 约束条件不足 | 明确禁止做事项 |

### 流程衔接问题

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| 设计文档看不懂 | Solution 到设计缺少过渡 | 增加设计 Agent 的上下文输入要求 |
| 代码与设计不符 | 设计文档不够详细 | 设计阶段输出伪代码级细节 |
| 测试用例覆盖不全 | 测试 Agent 缺少设计输入 | 确保测试 Agent 读取设计文档 |

### 产出物质量问题

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| PRD 需求模糊 | PM Agent 未充分挖掘需求 | 增加多轮对话确认环节 |
| Solution 缺少可行性分析 | 未识别技术约束 | 增加技术可行性检查点 |
| 设计文档过于复杂 | 未遵循"简单设计"原则 | 拆分模块，分阶段设计 |

## 4. 给出解决方案

针对识别出的问题，提供：
1. **即时解决方案** - 立即可以采取的措施
2. **长期改进方案** - 预防类似问题的机制
3. **需要修改的文件** - 具体需要调整的配置或文档

## 5. 执行修复（可选）

如用户同意，执行修复：
- 使用 `agent-optimize` 调整 Agent 提示词
- 使用 `skill-develop` 创建新的检查 Skill
- 更新相关产出物模板

# 诊断检查清单

分析问题时的标准检查项：

- [ ] 问题环节定位准确
- [ ] 相关产出物已读取
- [ ] 根因分析有理有据
- [ ] 解决方案具体可行
- [ ] 长期预防措施已考虑

# 常见诊断场景

## 场景一：Agent 输出不符合预期

**诊断步骤：**
1. 读取对应 Agent 文件
2. 检查 description 是否包含触发场景
3. 检查工作流程是否有检查清单
4. 检查约束条件是否明确

**修复方案：**
- 优化 Agent 提示词（调用 `agent-optimize`）
- 增加示例输出
- 明确禁止做事项

## 场景二：流程在某环节反复循环

**诊断步骤：**
1. 检查上一环节的产出物是否完整
2. 检查当前环节的输入要求是否清晰
3. 检查产出物标准是否过高或模糊

**修复方案：**
- 调整产出物验收标准
- 增加中间检查点
- 拆分过长的流程步骤

## 场景三：多个 Agent 职责冲突

**诊断步骤：**
1. 对比相关 Agent 的职责描述
2. 检查是否有重叠的"能做什么"
3. 检查衔接点的产出物定义

**修复方案：**
- 重新划分职责边界
- 明确产出物交接标准
- 增加专门的衔接 Agent
