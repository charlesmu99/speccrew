# 触发场景

- 开发 Agent 完成代码，用户请求启动测试
- 用户说"开始测试"、"验收测试"、"跑测试"

# 工作流程

## 步骤一：读取输入

1. PRD 验收标准：`projects/pXXX/01.prds/[功能名]-prd.md`（重点读"验收标准"章节）
2. 接口契约：`projects/pXXX/02.solutions/[功能名]-api-contract.md`
3. 开发任务记录：`projects/pXXX/04.tasks/[frontend|backend]/[功能名]-task.md`（了解偏差）
4. 测试规范：`.devcrew-workspace/knowledge/architecture/conventions/testing.md`

## 步骤二：准备测试用例

检查是否已有测试用例文件：`projects/pXXX/05.tests/cases/[功能名]-test-cases.md`

**如无则创建**，读取模板：
```
.qoder/skills/devcrew-test-report/templates/TEST-CASE-TEMPLATE.md
```

按 PRD 验收标准逐条拆解测试用例：

| 用例ID | 测试场景 | 前置条件 | 操作步骤 | 预期结果 |
|--------|----------|----------|----------|----------|
| AC-001 | [验收标准对应的场景] | [前置] | [步骤] | [期望] |

**覆盖要求**：
- 每条 P0 验收标准至少对应一个测试用例
- 包含正常流程和异常流程

## 步骤三：执行测试

逐条执行测试用例，记录实际结果：

| 用例ID | 测试场景 | 结果 | 缺陷ID | 备注 |
|--------|----------|------|--------|------|
| AC-001 | [场景] | ✅ 通过 | - | |
| AC-002 | [场景] | ❌ 失败 | BUG-001 | |

**发现 Bug 时**：
- 记录完整的复现步骤、预期结果、实际结果
- 分配严重程度（高/中/低）
- **不自行修改代码**，在报告中记录后继续测试剩余用例

## 步骤四：漂移检测（实现与设计对照）

读取详细设计文档：`projects/pXXX/03.designs/[frontend|backend]/[功能名]-design.md`

对照实际代码实现，检查是否存在**未申报的偏差**：

| 检查项 | 设计说明 | 实际实现 | 是否偏差 |
|------|---------|---------|--------|
| [接口路径/返回结构/业务逻辑等] | [设计要求] | [实际情况] | ✅ / ⚠️偏差 |

**检查策略**：
- 接口路径、请求参数、响应结构是否与接口契约一致
- 关键业务逻辑是否按设计实现，无自行扩展或简化
- 任务记录中标注的偏差是否已明确说明原因

**发现未申报偏差时**：在测试报告中单独列出，提要用户决定是否接受或回滚修正。

## 步骤五：生成测试报告

读取模板：`.qoder/skills/devcrew-test-report/templates/TEST-REPORT-TEMPLATE.md`

填写测试报告，包含：
- 总体结论（通过率、遗留缺陷数）
- 验收测试结果明细
- 缺陷清单（每个缺陷的完整描述）
- 遗留问题说明（哪些缺陷本次不修复及原因）

写入路径：`projects/pXXX/05.tests/reports/[功能名]-test-report.md`

## 步骤六：处理失败用例

如有测试失败：
1. 告知用户缺陷详情
2. 请用户决定：回溯 Dev Agent 修复 or 接受遗留
3. 修复完成后重新测试失败的用例
4. 更新测试报告

## 步骤七：请求上线确认

测试全部通过（或遗留问题已明确处理方案）后：

```
测试完成：
- 测试报告：projects/pXXX/05.tests/reports/[功能名]-test-report.md
- 通过率：[X/Y]（[XX%]）
- 遗留缺陷：[N] 条

请确认以下内容：
1. 测试结论是否可接受？
2. 遗留缺陷的处理方式是否认可？

⚠️ 确认无误后，功能将进入上线流程。
```

# 检查清单

- [ ] 已读取 PRD 验收标准，测试用例与验收标准一一对应
- [ ] 所有 P0 验收标准都有对应测试用例
- [ ] 测试用例已覆盖正常流程和主要异常流程
- [ ] 所有测试用例都已执行并记录结果
- [ ] 每个缺陷都有完整的复现步骤
- [ ] 已对照详细设计文档，检查未申报偏差
- [ ] 测试报告已生成并写入正确路径
- [ ] 已向用户展示测试摘要并请求上线确认
