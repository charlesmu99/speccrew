> যেকোনো সফ্টওয়্যার প্রকল্পের জন্য দ্রুত ইঞ্জিনিয়ারিং বাস্তবায়ন সক্ষম করে এমন একটি ভার্চুয়াল AI ডেভেলপমেন্ট টিম

## SpecCrew কী?

SpecCrew একটি এম্বেডেড ভার্চুয়াল AI ডেভেলপমেন্ট টিম ফ্রেমওয়ার্ক। এটি পেশাদার সফ্টওয়্যার ইঞ্জিনিয়ারিং ওয়ার্কফ্লো (PRD → Feature Design → System Design → Dev → Deployment → Test) কে পুনরায় ব্যবহারযোগ্য Agent ওয়ার্কফ্লোতে রূপান্তর করে, ডেভেলপমেন্ট টিমগুলিকে Specification-Driven Development (SDD) অর্জনে সাহায্য করে, বিশেষত বিদ্যমান প্রকল্পগুলির জন্য উপযুক্ত।

বিদ্যমান প্রকল্পগুলিতে Agent এবং Skill সংহত করে, টিমগুলি দ্রুত প্রকল্প ডকুমেন্টেশন সিস্টেম এবং ভার্চুয়াল সফ্টওয়্যার টিম চালু করতে পারে, মানক ইঞ্জিনিয়ারিং ওয়ার্কফ্লো অনুসরণ করে নতুন বৈশিষ্ট্য এবং পরিবর্তন ধাপে ধাপে বাস্তবায়ন করতে পারে।

## ✨ মূল বৈশিষ্ট্য

### 🏭 ভার্চুয়াল সফ্টওয়্যার টিম
এক ক্লিকে **7টি পেশাদার Agent ভূমিকা** + **30+টি Skill ওয়ার্কফ্লো** তৈরি, একটি সম্পূর্ণ ভার্চুয়াল সফ্টওয়্যার টিম গঠন:
- **Team Leader** - গ্লোবাল পরিকল্পনা এবং ইটারেশন ব্যবস্থাপনা
- **Product Manager** - প্রয়োজনীয়তা বিশ্লেষণ এবং PRD আউটপুট
- **Feature Designer** - বৈশিষ্ট্য ডিজাইন + API চুক্তি
- **System Designer** - Frontend/Backend/Mobile/Desktop সিস্টেম ডিজাইন
- **System Developer** - মাল্টিপ্ল্যাটফর্ম সমান্তরাল উন্নয়ন
- **Test Manager** - তিন-পর্যায়ের পরীক্ষা সমন্বয়
- **Task Worker** - সমান্তরাল উপ-কার্য সম্পাদন

### 📐 ISA-95 ছয়-পর্যায়ের মডেলিং
আন্তর্জাতিক **ISA-95** মডেলিং পদ্ধতির উপর ভিত্তি করে, ব্যবসায়িক প্রয়োজনীয়তা থেকে সফ্টওয়্যার সিস্টেমে রূপান্তরের মানকায়ন:
```
Domain Descriptions → Functions in Domains → Functions of Interest
     ↓                       ↓                      ↓
Information Flows → Categories of Information → Information Descriptions
```
- প্রতিটি পর্যায় নির্দিষ্ট UML ডায়াগ্রামের সাথে সামঞ্জস্যপূর্ণ (use case, sequence, class)
- ব্যবসায়িক প্রয়োজনীয়তা "ধাপে ধাপে পরিশোধিত" হয়, তথ্য ক্ষতি ছাড়াই
- আউটপুটগুলি উন্নয়নের জন্য সরাসরি ব্যবহারযোগ্য

### 📚 জ্ঞান ভিত্তি সিস্টেম
তিন-স্তরের জ্ঞান ভিত্তি আর্কিটেকচার নিশ্চিত করে যে AI সর্বদা "একক সত্যের উৎস" এর উপর ভিত্তি করে কাজ করে:

| স্তর | ডিরেক্টরি | বিষয়বস্তু | উদ্দেশ্য |
|------|-----------|-----------|----------|
| L1 সিস্টেম জ্ঞান | `knowledge/techs/` | টেক স্ট্যাক, আর্কিটেকচার, কনভেনশন | AI প্রকল্পের প্রযুক্তিগত সীমানা বোঝে |
| L2 ব্যবসায়িক জ্ঞান | `knowledge/bizs/` | মডিউল ফাংশন, ব্যবসায়িক প্রবাহ, সত্তা | AI ব্যবসায়িক যুক্তি বোঝে |
| L3 ইটারেশন আর্টিফ্যাক্ট | `iterations/iXXX/` | PRD, ডিজাইন ডকুমেন্ট, পরীক্ষার প্রতিবেদন | বর্তমান প্রয়োজনীয়তার জন্য সম্পূর্ণ ট্রেসেবিলিটি চেইন |

### 🔄 চার-পর্যায়ের জ্ঞান পাইপলাইন
**স্বয়ংক্রিয় জ্ঞান উৎপাদন আর্কিটেকচার**, সোর্স কোড থেকে ব্যবসায়িক/প্রযুক্তিগত ডকুমেন্টেশনের স্বয়ংক্রিয় উৎপাদন:
```
পর্যায় 1: সোর্স কোড স্ক্যান → মডিউল তালিকা তৈরি
পর্যায় 2: সমান্তরাল বিশ্লেষণ → বৈশিষ্ট্য বের করা (মাল্টি-ওয়ার্কার সমান্তরাল)
পর্যায় 3: সমান্তরাল সারাংশ → মডিউল ওভারভিউ সম্পূর্ণ করা (মাল্টি-ওয়ার্কার সমান্তরাল)
পর্যায় 4: সিস্টেম সংযোজন → সিস্টেম প্যানোরামা তৈরি
```
- **পূর্ণ সিঙ্কronization** এবং **ইনক্রিমেন্টাল সিঙ্কronization** সমর্থন করে (Git diff ভিত্তিক)
- একজন ব্যক্তি অপ্টিমাইজ করে, টিম শেয়ার করে

### 🔧 Harness বাস্তবায়ন কাঠামো
**স্ট্যান্ডার্ডাইজড এক্সিকিউশন ফ্রেমওয়ার্ক**, ডিজাইন ডকুমেন্টকে নির্ভুলভাবে এক্সিকিউটেবল ডেভেলপমেন্ট নির্দেশাবলীতে রূপান্তর নিশ্চিত করে:
- **অপারেশন ম্যানুয়াল নীতি**: Skill হল SOP, ধাপগুলি স্পষ্ট, ধারাবাহিক, স্বয়ংসম্পূর্ণ
- **ইনপুট-আউটপুট চুক্তি**: ইন্টারফেস স্পষ্টভাবে সংজ্ঞায়িত, সিউডোকোডের মতো কঠোরভাবে কার্যকর
- **গ্র্যাডুয়াল ডিসক্লোজার আর্কিটেকচার**: তথ্য স্তরে স্তরে লোড হয়, একবারে অত্যধিক প্রসঙ্গ এড়ায়
- **সাব-এজেন্ট ডেলিগেশন**: জটিল কাজ স্বয়ংক্রিয়ভাবে বিভক্ত হয়, সমান্তরাল কার্যকরীকরণ গুণমান নিশ্চিত করে

## ৮টি মূল সমস্যা সমাধান

### ১. AI বিদ্যমান প্রকল্প ডকুমেন্টেশন উপেক্ষা করে (জ্ঞান ব্যবধান)
**সমস্যা**: বিদ্যমান SDD বা Vibe Coding পদ্ধতিগুলি AI-এর রিয়েল-টাইমে প্রকল্প সারাংশ করার উপর নির্ভর করে, সহজেই গুরুত্বপূর্ণ প্রসঙ্গ মিস হয় এবং ডেভেলপমেন্ট ফলাফল প্রত্যাশা থেকে বিচ্যুত হয়।

**সমাধান**: `knowledge/` রিপোজিটরি প্রকল্পের "একমাত্র সত্যের উৎস" হিসাবে কাজ করে, আর্কিটেকচার ডিজাইন, কার্যকরী মডিউল এবং ব্যবসায়িক প্রক্রিয়াগুলি জমা করে নিশ্চিত করে যে প্রয়োজনীয়তাগুলি উৎস থেকে পথে থাকে।

### ২. PRD থেকে সরাসরি প্রযুক্তিগত ডকুমেন্টেশন (বিষয়বস্তু বাদ পড়া)
**সমস্যা**: PRD থেকে সরাসরি বিস্তারিত ডিজাইনে যাওয়া সহজেই প্রয়োজনীয়তার বিস্তারিত মিস হয়, বাস্তবায়িত বৈশিষ্ট্যগুলি প্রয়োজনীয়তা থেকে বিচ্যুত হয়।

**সমাধান**: প্রযুক্তিগত বিস্তারিত ছাড়াই শুধুমাত্র প্রয়োজনীয়তার কঙ্কালে ফোকাস করে **Feature Design ডকুমেন্ট** পর্যায় প্রবর্তন করুন:
- কোন পৃষ্ঠা এবং উপাদান অন্তর্ভুক্ত?
- পৃষ্ঠা অপারেশন প্রবাহ
- ব্যাকেন্ড প্রক্রিয়াকরণ লজিক
- ডেটা স্টোরেজ কাঠামো

ডেভেলপমেন্ট শুধুমাত্র নির্দিষ্ট টেক স্ট্যাকের উপর ভিত্তি করে "মাংস পূরণ করতে" হবে, নিশ্চিত করে যে বৈশিষ্ট্যগুলি "হাড়ের কাছাকাছি (প্রয়োজনীয়তা)" বৃদ্ধি পায়।

### ৩. Agent সার্চ সুযোগ অনিশ্চিত (অনিশ্চয়তা)
**সমস্যা**: জটিল প্রকল্পগুলিতে, AI-এর বিস্তৃত কোড এবং ডকুমেন্ট সার্চ অনিশ্চিত ফলাফল দেয়, সামঞ্জস্যতা নিশ্চিত করা কঠিন করে তোলে।

**সমাধান**: প্রতিটি Agent-এর প্রয়োজনের উপর ভিত্তি করে ডিজাইন করা স্পষ্ট ডকুমেন্ট ডিরেক্টরি কাঠামো এবং টেমপ্লেট, নিশ্চিততা নিশ্চিত করতে **ধাপে ধাপে প্রকাশ এবং চাহিদা অনুযায়ী লোডিং** বাস্তবায়ন করে।

### ৪. ধাপ এবং কাজ অনুপস্থিত (প্রক্রিয়া বিচ্ছেদ)
**সমস্যা**: সম্পূর্ণ ইঞ্জিনিয়ারিং প্রক্রিয়া কভারেজের অভাব সহজেই গুরুত্বপূর্ণ ধাপ মিস করে, গুণমান নিশ্চিত করা কঠিন করে তোলে।

**সমাধান**: সম্পূর্ণ সফ্টওয়্যার ইঞ্জিনিয়ারিং জীবনচক্র কভার করুন:
```
PRD (প্রয়োজনীয়তা) → Feature Design (বৈশিষ্ট্য ডিজাইন) → API Contract (চুক্তি)
    → System Design (সিস্টেম ডিজাইন) → Dev (ডেভেলপমেন্ট) → Deployment (স্থাপনা) → Test (পরীক্ষা)
```
- প্রতিটি পর্যায়ের আউটপুট পরবর্তী পর্যায়ের ইনপুট
- প্রতিটি ধাপ এগিয়ে যাওয়ার আগে মানব নিশ্চিতকরণ প্রয়োজন
- সমস্ত Agent কার্যকরীকরণে সম্পন্ন হওয়ার পরে স্ব-পরীক্ষা সহ todo তালিকা আছে

### ৫. কম টিম সহযোগিতা দক্ষতা (জ্ঞান সিলো)
**সমস্যা**: AI প্রোগ্রামিং অভিজ্ঞতা টিমগুলির মধ্যে ভাগ করা কঠিন, পুনরাবৃত্ত ত্রুটির দিকে নিয়ে যায়।

**সমাধান**: সমস্ত Agent, Skill এবং সম্পর্কিত ডকুমেন্ট সোর্স কোডের সাথে ভার্সন কন্ট্রোল করা হয়:
- একজনের অপ্টিমাইজেশন টিম দ্বারা ভাগ করা হয়
- জ্ঞান কোড বেসে জমা হয়
- টিম সহযোগিতা দক্ষতা উন্নত হয়

### ৭. একক Agent প্রসঙ্গ খুব দীর্ঘ (পারফরম্যান্স বাধা)
**সমস্যা**: বড় জটিল কাজগুলি একক Agent প্রসঙ্গ উইন্ডো অতিক্রম করে, বোঝাপড়া বিচ্যুতি এবং আউটপুট গুণমান হ্রাস ঘটায়।

**সমাধান**: **Sub-Agent অটো-ডিসপ্যাচ মেকানিজম**:
- জটিল কাজগুলি স্বয়ংক্রিয়ভাবে চিহ্নিত হয় এবং সাব-টাস্কে বিভক্ত হয়
- প্রতিটি সাব-টাস্ক বিচ্ছিন্ন প্রসঙ্গ সহ একটি স্বাধীন Sub-Agent দ্বারা কার্যকর করা হয়
- প্যারেন্ট Agent সমন্বয় এবং একত্রিত করে সামগ্রিক সামঞ্জস্য নিশ্চিত করে
- একক Agent প্রসঙ্গ সম্প্রসারণ এড়িয়ে আউটপুট গুণমান নিশ্চিত করে

### ৮. প্রয়োজনীয়তা পুনরাবৃত্তি বিশৃঙ্খলা (ব্যবস্থাপনা অসুবিধা)
**সমস্যা**: একই শাখায় মিশ্রিত একাধিক প্রয়োজনীয়তা একে অপরকে প্রভাবিত করে, ট্র্যাকিং এবং রোলব্যাক কঠিন করে তোলে।

**সমাধান**: **প্রতিটি প্রয়োজনীয়তা একটি স্বাধীন প্রকল্প হিসাবে**:
- প্রতিটি প্রয়োজনীয়তা একটি স্বাধীন পুনরাবৃত্তি ডিরেক্টরি তৈরি করে `iterations/iXXX-[প্রয়োজনীয়তা-নাম]/`
- সম্পূর্ণ বিচ্ছিন্নতা: ডকুমেন্ট, ডিজাইন, কোড এবং পরীক্ষা স্বাধীনভাবে পরিচালিত
- দ্রুত পুনরাবৃত্তি: ছোট গ্রানুলারিটি ডেলিভারি, দ্রুত যাচাইকরণ, দ্রুত স্থাপন
- নমনীয় আর্কাইভিং: সম্পন্ন হওয়ার পরে, স্পষ্ট ঐতিহাসিক ট্রেসেবিলিটি সহ `archive/`-এ আর্কাইভ করুন

### ৬. ডকুমেন্ট আপডেট বিলম্ব (জ্ঞান ক্ষয়)
**সমস্যা**: প্রকল্পগুলি বিকশিত হওয়ার সাথে সাথে ডকুমেন্টগুলি পুরানো হয়ে যায়, AI ভুল তথ্য নিয়ে কাজ করে।

**সমাধান**: Agent-এর স্বয়ংক্রিয় ডকুমেন্ট আপডেট ক্ষমতা আছে, জ্ঞান বেস সঠিক রাখতে রিয়েল-টাইমে প্রকল্প পরিবর্তনগুলি সিঙ্ক্রোনাইজ করে।
